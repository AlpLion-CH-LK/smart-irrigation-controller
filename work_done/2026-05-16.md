## Design Decisions
- **No bins** — bins were only needed for Q-learning's lookup table; PPO neural network takes raw floats directly
- **Temperature max = 40°C** — reflects real local conditions (not the 45°C generic default)
- **`hour / 24.0`** — simple normalization chosen over sin/cos encoding; sufficient for irrigation patterns
- **Both `growth_stage` and `current_day` in obs** — stage gives coarse context (which phase), day gives fine context (how far into that phase)
- **`planting_date` from system clock** — no extra sensor needed; farmer enters once at setup

## Stage 2 — Step-based simulation (`src/irrigation/sensors/simulation.py`)

### Problem fixed
`SimulatedSoilMoistureSensor` was using real wall-clock elapsed time for ET:
```python
elapsed_hours = (now - self._last_read).total_seconds() / 3600.0
```
During training (1000s steps/sec), real elapsed time ≈ microseconds → no ET → moisture never drained → agent learned nothing realistic.

Also: soil sensor only returned `soil_moisture_pct`; environment now needs `temperature_celsius`, `humidity_pct`, `is_raining` too — no combined sensor existed.

### Changes

**`SimulatedSoilMoistureSensor`**
- Removed wall-clock ET logic and `_last_read` tracking
- Added `step_hours: float = 0.5` — applies exactly `et_rate_per_hour × step_hours` ET per `read()` call
- Added `_simulated_hour` counter → `timestamp` reflects simulated time, not real time
- Added `reset(initial_moisture_pct, initial_hour)` for episode resets

**`SimulatedWeatherSensor`**
- Added `step_hours: float = 0.5` and `_simulated_hour` counter
- Temperature diurnal cycle now uses `_simulated_hour` instead of `datetime.now()`
- Added `reset(initial_hour)` for episode resets

**`CombinedSimulatedSensor` (new class)**
- Wraps both sensors, merges into one `SensorReading`
- Applies rain to soil moisture automatically when it rains
- Has `reset()` to restart both sensors for a new episode
- This is what gets passed to `IrrigationEnvironment`

### Usage
```python
soil = SimulatedSoilMoistureSensor(initial_moisture_pct=60.0, step_hours=0.5)
weather = SimulatedWeatherSensor(step_hours=0.5)
sensor = CombinedSimulatedSensor(soil, weather)
actuator = SimulatedActuator(soil_sensor=soil)
env = IrrigationEnvironment(sensor, actuator, ChiliProfile(), planting_date=...)
```

## Stage 4 — Continuous action space + Zone-based water calibration

### Root cause fixed
Discrete actions (4 fixed buckets) cannot match stage-specific water needs.
Chili needs 5L/day at germination vs 12.7L/day at flowering — no fixed bucket fits both.
Also: `moisture_per_litre=1.5` default was **15× too high** for a 3m² zone (should be 0.1).

### New file: `src/irrigation/zone_config.py`
`ZoneConfig` dataclass — defaults match Sri Lanka DoA Jaffna chilli recommendation:
- `area_m2=3.0` (3m × 1m raised bed), `irrigation_type="drip"` (90% efficiency)
- `max_litres_per_event=15.0`, `emergency_min_litres=2.0`, `root_depth_m=0.3`
- `moisture_per_litre` = `(efficiency / (area × depth × 1000)) × 100` = **0.1 %/L**

- Added `_STAGE_ETC_LITRES_PER_DAY = [5.0, 8.6, 12.7, 12.2, 10.3]`

### `actuators/base.py` — Discrete → Continuous
- Removed `IrrigationAction` enum, `ACTION_DURATIONS`, `FLOW_RATE_LPM`
- New `IrrigationCommand(water_litres: float)` — single field

### `actuators/simulation.py`
- Fixed `moisture_per_litre` default: `1.5 → 0.1` (15× calibration fix)

### `actuators/valve.py`
- Duration = `water_litres / flow_rate_lpm × 60` (added `flow_rate_lpm=2.0` param)

### `rl/environment.py`

### `rl/reward.py`

## Stage 6 — Curriculum Learning (two-phase PPO training)

### What changed

**`src/irrigation/rl/gym_env.py`**
- Added `training_phase: int = 1` parameter
- Added `_PHASE1_CONDITIONS` (dry months: Feb–Apr, 31–33°C, 62–67% humidity, no rain)
- Added `_PHASE2_CONDITIONS` (all months: dry + moderate + monsoon including rainy season)
- `reset()` randomly picks weather conditions each episode based on phase
- Weather conditions (base_temp, base_humidity, is_rainy_season) updated per episode

**`scripts/train.py`**
- Restructured into two-phase training loop:
  - Phase 1 (40% of timesteps): favorable dry months → agent learns basic irrigation
  - Phase 2 (60% of timesteps): all months → agent generalises to monsoon + varied conditions
- Phase 1 model saved to `models/ppo_phase1_complete`
- Final model saved to `models/ppo_irrigation_final`
- Added `--phase1-ratio` CLI argument (default 0.4)

### Why curriculum learning
Starting with monsoon months confuses the agent — it can skip irrigation and still get the rain bonus. Starting with dry months forces it to actively irrigate and learn the core skill first.

## Stage 7 — Plant Health Tracker

### New file: `src/irrigation/rl/health_tracker.py`
`PlantHealthTracker` class — tracks cumulative plant health across one episode.

**Per step classification:**
- `optimal_hours` — moisture in stage-specific optimal range (full growth)
- `stress_hours` — moisture below stress threshold (impaired growth)
- `critical_hours` — moisture below wilting point (no growth, damage)
- `waterlog_hours` — moisture above field capacity (root damage)

**Per stage health score:**
- `StageHealthRecord` per stage — `stage_0_health` to `stage_4_health` (0.0–1.0)
- Reported at episode end via `summary()` dict

**Observation change:**
- `health_score` added as 8th observation value (index 7)
- Observation shape: `(7,) → (8,)`
- Agent can see its plant's accumulated health and self-correct

**Also fixed:**
- `SimulatedSoilMoistureSensor` ET rate: `2.0 → 0.083 %/hr`
  - Correct for Jaffna ET₀ ≈ 6mm/day on 3m² zone with 0.3m root depth

## Stage 8 — Dynamic Stage Progression

### New file: `src/irrigation/rl/dynamic_stage.py`
`DynamicStageTracker` class — replaces fixed calendar-based stage progression.

**Stage advance requires BOTH:**
1. Minimum hours elapsed in this stage
2. Health ratio >= required threshold

**If health never reaches threshold — forced advance at max hours (plant stunted but alive).**

**Stage timing configs (hours):**

| Stage | Min hours | Max hours | Health required |
|---|---|---|---|
| 0 Germination | 360 (15d) | 720 (30d) | 70% |
| 1 Vegetative | 720 (30d) | 1320 (55d) | 70% |
| 2 Flowering | 600 (25d) | 960 (40d) | 75% |
| 3 Fruit dev | 600 (25d) | 960 (40d) | 70% |
| 4 Maturity | 600 (25d) | 960 (40d) | 65% |

**Health accumulation per step:**
- Moisture in optimal range → +1.0
- Moisture in stress range → +0.5 (slow growth)
- Below stress threshold → +0.0 (no growth)

**`src/irrigation/rl/environment.py`**
- Added `_sim_stage: int | None = None` attribute (same pattern as `_sim_day`)
- `_growth_stage()` returns `_sim_stage` when set — dynamic stage used by reward function

**`src/irrigation/rl/gym_env.py`**
- Sets `_env._sim_stage` before each step so reward uses dynamic stage
- `info` dict reports both `calendar_stage` and `dynamic_stage` each step
- Episode summary includes `stage_days_taken` and `stage_delay_days`

**Verified behaviour:**
- Healthy plant: Stage 0 → Stage 1 at day 20 (matches calendar)
- Stressed plant: Stage 0 forced at day 29 (+9 days late), Stage 1 forced at day 84 (+24 days late)

## Stage 9 — Plant Death Condition (all stages)

### New file: `src/irrigation/rl/vitality.py`
`PlantVitalityTracker` class — tracks plant survival using stage-specific critical thresholds.

**Vitality score (0.0 → 1.0):**
- 1.0 = perfectly healthy
- 0.5 = stressed but alive
- 0.0 = dead → episode terminates immediately

**Stage-specific death configs:**

| Stage | Dry threshold | Dry limit | Wet limit | Vitality drain/hr |
|---|---|---|---|---|
| 0 Germination | wilting_point (20%) | 2 hrs | 4 hrs | 0.20 |
| 1 Vegetative | wilting_point (20%) | 6 hrs | 8 hrs | 0.12 |
| 2 Flowering | **stress_threshold (55%)** | 12 hrs | 10 hrs | 0.08 |
| 3 Fruit dev | wilting_point (25%) | 12 hrs | 12 hrs | 0.06 |
| 4 Maturity | wilting_point (20%) | 24 hrs | 24 hrs | 0.03 |

**Stage 2 is special:** dry threshold = stress_threshold (55%) because flower drop at that moisture level = total crop failure, even if the plant physically survives.

**Death penalties (added to reward on termination):**
- Stage 0: −100 (entire season wasted)
- Stage 1: −80
- Stage 2: −60 (total crop failure)
- Stage 3: −40
- Stage 4: −20 (some crop may be salvageable)

**Death causes tracked:** `drought` or `waterlogging`

**Observation change:**
- `vitality` added as 9th observation value (index 8)
- Observation shape: `(8,) → (9,)`
- Agent can see plant survival status and act before it is too late

**`src/irrigation/rl/gym_env.py`**
- `PlantVitalityTracker.update()` called each step with current dynamic stage
- Death penalty added to reward when `is_dead = True`
- `terminated = True` immediately on plant death
- Episode summary includes `plant_dead`, `death_cause`, `death_stage`

**Verified behaviour:**
- Stage 0 drought death: vitality 1.0 → 0.8 → 0.0 within 2 steps (−100 penalty)
- Healthy plant with adequate irrigation: vitality stays 1.0 throughout
